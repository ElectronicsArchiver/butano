Logging module

Contents

  • Reference
    • Files
    • Defines

Butano's logging system.

It allows to print information on the console of multiple emulators.

It can be enabled or disabled by overloading the definition of BTN_CFG_LOG_ENABLED .

It supports printing on only one emulator at once. The supported emulator can be changed by overloading the definition of BTN_CFG_LOG_BACKEND .

Specifies if emulator logging is enabled or not.

Values not specified in BTN_LOG_BACKEND_* macros are not allowed.

#define BTN_CFG_LOG_MAX_SIZE
#include <btn_config_log.h>

Specifies the maximum number of characters that can be logged with one BTN_LOG call.

#define BTN_LOG(...)
#include <btn_log.h>

Prints in one line of text the representation of the given parameters.

Example:

BTN_LOG("Invalid integer: ", integer);

Custom parameter types are supported by overloading btn::ostringstream::operator<<.

Example:

btn::ostringstream& operator<<(btn::ostringstream& stream, const custom_type& custom_value)
{
    stream.append("custom_type: ");
    stream.append(custom_value.data);
    return stream;
}
